### Шаг 3: Тестировать восстановление

```python
import torch
from model.aotgan import HybridInpaintGenerator
from torchvision.utils import save_image

# Загрузить обученную модель
model = HybridInpaintGenerator(args)
model.load_state_dict(torch.load('checkpoints/G0500000.pt'))
model.eval()

# Подготовить данные
image = torch.randn(1, 3, 256, 256)  # Ваше изображение
mask = torch.bernoulli(torch.full((1, 1, 256, 256), 0.2))  # Маска повреждения

# Восстановление
with torch.no_grad():
    restored, confidence = model(image, mask)

# Сохранить результаты
save_image((restored + 1) / 2, 'restored.png')
save_image(confidence, 'confidence.png')
```

### Генератор
```python
# БЫЛО (базовый AOT-GAN):
pred_img = model(image, mask)  # → (B, 3, H, W)

# СТАЛО (HFI-Gen):
restored, confidence = model(image, mask)
# restored: (B, 3, H, W) ∈ [-1, 1]
# confidence: (B, 1, H, W) ∈ [0, 1]
```

## 🔍 АРХИТЕКТУРА В ДВУХ СЛОВАХ

HFI-Gen = U-Net (для деталей) + AOT (для контекста) + Confidence (для честности)

```
INPUT (RGB + Mask)
    ↓
[ENCODER 4 уровня]  ← Более детальный чем базовый AOT
    ↓
[BOTTLENECK с AOT] ← Контекстная агрегация
    ↓
[DECODER + SKIP-CONNECTIONS] ← Основное восстановление
    ↓
[CONFIDENCE BRANCH] ← Параллельный decoder для уверенности
    ↓
(RESTORED IMAGE, CONFIDENCE MAP)
```

## 🧠 ПОЧЕМУ ЭТО РАБОТАЕТ ДЛЯ СУДЕБНОЙ ЭКСПЕРТИЗЫ

1. **4-уровневый encoder**: Сохраняет мелкие детали (текст, микроструктуры)
2. **Skip-connections**: Напрямую передают низкоуровневую информацию
3. **AOT блоки**: Агрегируют глобальный контекст
4. **Confidence branch**: Показывает, где модель неуверена
   → Судья видит: "Тут я уверена 0.95, а тут 0.45"
   → Позволяет оценить допустимость доказательства

## 🐛 ЕСЛИ ЧТО-ТО НЕ РАБОТАЕТ

### "ImportError: cannot import name 'MultiScaleSSIM'"
→ Убедитесь что используете обновленный loss/loss.py

### "TypeError: forward() got unexpected argument"
→ Проверьте что loss функции инициализированы с device параметром:
  loss = MultiScaleSSIM(device='cuda')

### "Generator returns tuple, expected tensor"
→ Trainer уже поддерживает оба типа! Проверьте версию trainer.py

### Confidence map всегда ~0.5 (не обучается)
→ Увеличьте conf_reg_weight до 0.5-1.0 и больше итераций

### Нужна помощь?
→ Читайте раздел "Диагностика проблем" в HFI_GEN_USAGE.md
