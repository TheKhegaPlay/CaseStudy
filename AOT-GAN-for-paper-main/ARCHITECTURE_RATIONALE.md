## Гибридная архитектура

### Проблема с оригинальным AOT-GAN

Оригинальный AOT-GAN (`InpaintGenerator` в исходном коде) использует **простую последовательную архитектуру**:

```
Input (RGB + mask, 4 каналов)
  ↓ encoder (3 Conv с downsampling)
  ↓ middle (AOT блоки)  
  ↓ decoder (2 UpConv)
  ↓ Output (RGB)
```

**Проблемы для судебной экспертизы:**
1. **Потеря мелких деталей**: Трёхуровневый encoder приводит к потере низкоуровневых признаков (текст высокого разрешения, микроструктуры, шумы).
2. **Отсутствие skip-connections**: Информация из ранних слоёв не используется в decoder, что приводит к артефактам на границах восстановленной области.
3. **Нет информации об уверенности**: Модель выдаёт одно изображение без указания того, в каких регионах восстановление надежно.

### Наше решение: Truncated U-Net (4 уровня)

```
Input (4 каналов)
  ↓
[ENCODER]
e1 (64 каналов, H×W)
  ↓ stride=2
e2 (128 каналов, H/2×W/2)
  ↓ stride=2  
e3 (256 каналов, H/4×W/4)
  ↓ stride=2
e4 (512 каналов, H/8×W/8) ─ BOTTLENECK

[BOTTLENECK с AOT]
m = AOTBlock(e4)  ← Агрегация контекста

[DECODER с SKIP-CONNECTIONS]
d4 = UpConv(m) ⊕ e3  ← concat с e3
  ↓
d3 = UpConv(d4) ⊕ e2  ← concat с e2
  ↓
d2 = UpConv(d3) ⊕ e1  ← concat с e1
  ↓
Output = Restored Image

[CONFIDENCE BRANCH (параллельно)]
c4 = UpConv(m) ⊕ e3
c3 = UpConv(c4) ⊕ e2
c2 = UpConv(c3) ⊕ e1
ConfidenceMap = Sigmoid(c2)
```

### Почему 4 уровня?

| Аспект | 3 уровня (оригинал) | 4 уровня (наш выбор) | 5+ уровней |
|--------|-------------------|-------------------|-----------|
| **Receptive field** | 64×64 | 128×128 | 256×256+ |
| **Сохранение деталей** | ❌ Плохо | ✅ Хорошо | ⚠️ Риск размытия |
| **Вычислительная сложность** | Мало | Оптимально | Много |
| **Для forensic images** | Недостаточно | ✅ **Идеально** | Чрезмерно |

**Научное обоснование:**
- Forensic изображения содержат информацию на разных масштабах:
  - **Уровень 1-2** (полное разрешение): текст, надписи, края объектов
  - **Уровень 3-4** (среднее разрешение): структура объектов, соотношение элементов
  - **Уровень 5+** (низкое разрешение): глобальная структура (редко нужна для судебных доказательств)
- 4 уровня обеспечивают баланс между контекстом и деталями.

## Усиленные skip-connections

### Механизм

В нашей архитектуре **каждый уровень decoder получает skip-соединение от соответствующего уровня encoder**:

```python
# Уровень 4
d4 = UpConv(m, 512→256)           # (B, 256, H/4, W/4)
d4 = ConvBlock(cat([d4, e3]))     # e3 имеет размер (B, 256, H/4, W/4)

# Уровень 3  
d3 = UpConv(d4, 256→128)          # (B, 128, H/2, W/2)
d3 = ConvBlock(cat([d3, e2]))     # e2 имеет размер (B, 128, H/2, W/2)

# Уровень 2
d2 = UpConv(d3, 128→64)           # (B, 64, H, W)
d2 = ConvBlock(cat([d2, e1]))     # e1 имеет размер (B, 64, H, W)
```

### Почему это критично для судебной экспертизы?

#### 1. **Сохранение текстовых элементов**

**Задача**: Восстановить повреждённое постановление суда с текстом.

- **Без skip-connections**: Encoder сжимает RGB информацию в bottleneck. При раскодировании модель должна "угадать" точное размещение и форму каждого символа. Результат: размытый, нечитаемый текст.

- **С skip-connections**: Encoder сохраняет точные границы и контуры символов на e1. При декодировании конкатенируем с e1, и модель легко восстанавливает точную форму символов. Результат: читаемый текст.

#### 2. **Сохранение мелких структур (трещины, царапины, следы)**

Forensic исследование часто зависит от микро-особенностей:
- Узоры отпечатков пальцев
- Царапины на документах (признак подделки)
- Тонкие линии и волосяные линии
- Зёрна фотошума (доказательство подлинности)

**Skip-connections обеспечивают:**
- Прямой путь для низкоуровневых признаков (текстур, краёв)
- Защиту от чрезмерного сглаживания (smoothing) в bottleneck
- Честное воспроизведение оригинальной текстуры повреждённой области

#### 3. **Уменьшение артефактов на границах маски**

**Проблема**: При отсутствии skip-connections граница между восстановленной и оригинальной областью часто видна (артефакт "seam").

**Skip-connections решают**: Информация о структуре вне маски напрямую передается в соответствующие уровни decoder, обеспечивая гладкий переход.

#### 4. **Математическое объяснение (Gradient Flow)**

В глубоких U-Net архитектурах skip-connections создают **несколько путей для обратного распространения** градиентов:

$$\nabla L = \nabla L_{\text{direct}} + \nabla L_{\text{skip-1}} + \nabla L_{\text{skip-2}} + \ldots$$

Это обеспечивает:
- Лучший gradient flow в глубокие слои
- Более стабильное обучение (меньше vanishing gradients)
- Более быструю сходимость

## Confidence Estimation Branch

### Проблема: "Слепая" генерация

Базовый GAN-based inpainting дает одно выходное изображение, но **не указывает уровень уверенности в каждом пикселе**. 

**Судебные последствия:**
- Эксперт видит восстановленное изображение, но не знает, в каких регионах оно надежно.
- При допросе в суде: "Докажите, что это восстановление не является галлюцинацией модели!" — нет объективного ответа.
- Восстановленные пиксели могут быть приняты судом как "почти оригинальные", но без confidence можно ошибиться на 50%.

### Наше решение: Параллельный Decoder для Confidence Map

```python
# Confidence Branch (отдельный от основного decoder)
c4 = UpConv(m) ⊕ e3
c3 = UpConv(c4) ⊕ e2
c2 = UpConv(c3) ⊕ e1
confidence_map = Sigmoid(Conv(c2))  # Выход [0, 1]
```

**Архитектурные особенности confidence branch:**
1. **Независимые параметры**: Не связана с основным decoder, может обучаться независимо.
2. **Используются те же skip-соединения**: Информация о структуре одинакова.
3. **Без dropout в confidence branch**: Дает стабильную, детерминированную оценку уверенности (в main decoder есть dropout для регуляризации).
4. **Sigmoid выход**: Выход в диапазон [0, 1], где 0 = не уверена, 1 = очень уверена.

### Научное обоснование

#### 1. **Bayesian Interpretation**

Confidence map можно интерпретировать как **approximation к posterior uncertainty**:

$$p(\text{confidence} \mid \text{observed features}, \theta) \approx \text{sigmoid}(f_{\text{conf}}(e_1, e_2, e_3))$$

Где $f_{\text{conf}}$ — confidence branch.

**В судебном контексте:** Это дает численное выражение для "уверенности эксперта" в восстановленной области.

#### 2. **Selective Reconstruction**

Не все регионы одинаково восстанавливаются. Confidence branch позволяет:
- **Высокая confidence**: Модель восстановила хорошо (текстура похожа, границы гладкие).
- **Низкая confidence**: Модель неуверена (слишком много неоднозначности, недостаточный контекст).

**Судебное применение:**
```
LOW CONFIDENCE REGION → "Требуется повторное исследование"
HIGH CONFIDENCE REGION → "Может быть использовано как доказательство"
```

#### 3. **Adversarial Robustness**

Confidence branch предотвращает модель от **ложной самоуверенности** (overconfidence):
- При обучении модель штрафуется за низкую confidence в правильно восстановленных регионах.
- Но также штрафуется за высокую confidence в НЕПРАВИЛЬНО восстановленных регионах.
- Результат: честная оценка уверенности.

## Loss Function Strategy

### Комбинированная Loss для HFI-Gen

```python
L_total = λ₁·L_L1 + λ₂·L_MS-SSIM + λ₃·L_Perceptual + λ₄·L_Style + 
          λ₅·L_Adversarial + λ₆·L_ConfidenceReg + λ₇·L_WeightedRecon
```

Где:
- **λ₁ = 1.0**: L1 loss (пиксельная точность)
- **λ₂ = 0.5**: Multi-Scale SSIM (структурное сходство)
- **λ₃ = 0.1**: Perceptual loss (VGG features)
- **λ₄ = 0.05**: Style loss (текстурное сходство)
- **λ₅ = 0.1**: Adversarial loss (натуральность)
- **λ₆ = 0.3**: Confidence Regularization (честная уверенность)
- **λ₇ = 0.2**: Weighted Reconstruction (weighted by confidence)

### 1. Multi-Scale SSIM (MS-SSIM)

#### Почему SSIM лучше чем простой L1 для судебных изображений?

**L1 loss проблема:**
```
Оригинал:   █████████ (текст чёрный)
L1 минимум: ░░░░░░░░░ (серый - минимизирует |черный - серый|)
```

L1 минимизирует пиксельное расстояние, но не структуру! Для текста это приводит к размытому серому цвету вместо чёткого чёрного.

**SSIM решение:**
```
SSIM = (2μₓμᵧ + C₁)(2σₓᵧ + C₂) / ((μₓ² + μᵧ² + C₁)(σₓ² + σᵧ² + C₂))
```

Вычисляет структурное сходство:
- $\mu$ — локальное среднее (яркость)
- $\sigma$ — локальная дисперсия (контраст)
- $\sigma_{xy}$ — ковариация (структура)

**Для судебных изображений:**
- Сохраняет четкость текста и краёв
- Восстанавливает истинные значения яркости
- Минимизирует размытие

#### Multi-Scale

MS-SSIM применяет SSIM на **разных масштабах** (full resolution, ×2 downsampling, ×4, ×8, ×16):

```
Уровень 1 (полное разрешение): детали текста, микро-структуры
Уровень 2: локальные паттерны, объекты
Уровень 3: глобальная структура, объекты
Уровень 4: очень грубая глобальная структура
Уровень 5: суперглобальная структура
```

**Судебная релевантность:**
- Уровни 1-2: Критичны для доказательств (текст, следы)
- Уровни 3-5: Важны для контекста (где находится повреждение, какой документ)

### 2. Perceptual Loss (VGG19 Features)

```python
L_Perceptual = Σ ||VGG(restored)_i - VGG(target)_i|| / n_features
```

**Почему нужен:**
- L1 + MS-SSIM работают с пиксельным пространством
- Perceptual loss работает с **semantic space** (смыслом содержимого)

**Пример:**
```
Оригинал: красная помада на лице
L1 loss может минимизировать ||красный - синий||, потому что они близко в RGB
Perceptual loss: VGG знает, что это "лицо" и "помада", не путает цвета
```

**Для судебных доказательств:**
- Восстановленный объект должен иметь правильное **семантическое значение**
- Восстановленная подпись должна выглядеть как подпись, а не случайные штрихи
- Восстановленное лицо должно быть узнаваемо

### 3. Confidence Regularization Loss

#### Формулировка

```python
L_ConfidenceReg = λ_valid · E[(1 - confidence) | valid_mask] + 
                  λ_masked · E[(1 - confidence) | inpaint_mask]
```

Где:
- `valid_mask = 1 - inpaint_mask` (целые, неповреждённые области)
- `inpaint_mask` (повреждённые области, требующие восстановления)

#### Интерпретация

**Логика:**
1. В **целых областях** (valid_mask = 1):
   - Confidence ДОЛЖНА быть высокой (модель видит оригинальный пиксель)
   - Штрафуем: $(1 - \text{confidence})$ — если confidence низкая, штраф высокий

2. В **повреждённых областях** (inpaint_mask = 1):
   - Confidence ДОЛЖНА быть высокой ЕСЛИ восстановление хорошее
   - Штрафуем: $(1 - \text{confidence})$ — если confidence низкая, штраф высокий
   - Но это штрафуется МЕНЬШЕ (λ_masked < λ_valid), потому что восстановление всегда неточно

#### Судебное значение

```
Результат:
- Целые регионы: confidence ≈ 1.0 (мы точно знаем оригинальное значение)
- Хорошо восстановленные регионы: confidence ≈ 0.8-0.9
- Плохо восстановленные регионы: confidence ≈ 0.3-0.5

Эксперт может сказать:
"В этих регионах я уверен (confidence > 0.85)"
"В этих регионах неопределённость (confidence < 0.5)"
```

### 4. Weighted Reconstruction Loss

```python
L_WeightedRecon = E[(1 + α·confidence) · |restored - target| · inpaint_mask]
```

**Логика:**
- Регионы с **высокой confidence** штрафуются СИЛЬНЕЕ за ошибки
- Регионы с **низкой confidence** штрафуются СЛАБЕЕ (модель "честна" в неуверенности)
- Это побуждает модель:
  - Быть точной где она уверена
  - Быть осторожной где неуверена

**Судебное значение:**
- Модель не может просто выдать средний результат (высокая confidence)
- Модель вынуждена либо:
  - Восстановить хорошо (высокая confidence + низкая ошибка)
  - Признать неопределённость (низкая confidence)

## Отличия от базового AOT-GAN

| Аспект | AOT-GAN (оригинал) | HFI-Gen (наша модель) | Преимущество |
|--------|-------------------|----------------------|-------------|
| **Encoder глубина** | 3 уровня | 4 уровня | Лучше сохраняет детали |
| **Skip-connections** | Нет | Да (e1, e2, e3) | Четче текст и грани |
| **Confidence output** | Нет | Да (parallel branch) | Видимость уверенности |
| **Loss function** | Adversarial + L1 | Adversarial + L1 + MS-SSIM + Perceptual + Confidence | Лучше структура + честность |
| **Dropout в decoder** | Нет | Да (0.3) | Больше регуляризации |
| **Confidence-aware loss** | Нет | Да | Взвешенное восстановление |

## Судебная релевантность

### Требования к судебным доказательствам (Legal Standards)

Согласно международным стандартам (Daubert standard, UK Civil Procedure Rules):
1. **Repeatability** — результат должен быть воспроизводимым
2. **Falsifiability** — метод должен быть проверяемым и опровергаемым
3. **Peer review** — метод принят в научном сообществе
4. **Error rate** — должна быть известна вероятность ошибки
5. **Relevance** — результат должен быть релевантен делу

### Как HFI-Gen удовлетворяет этим требованиям

#### 1. **Repeatability**
- Deterministic model (одних и тех же входов → одних и тех же выходов при inference с `model.eval()`)
- Confidence map позволяет повторить анализ и проверить консистентность

#### 2. **Falsifiability**
- Confidence map можно сравнить с другими методами восстановления
- Низкая confidence в регионах, где восстановление неправильно, свидетельствует о честности модели
- Можно провести тесты на синтетических повреждениях с известным оригиналом

#### 3. **Peer Review**
- Модель основана на AOT-GAN (уже опубликовано в конференциях)
- Архитектура использует стандартные компоненты (U-Net, skip-connections, VGG features)
- MS-SSIM и перцептуальные лоссы стандартны в CV

#### 4. **Error Rate**
- Confidence map дает численное выражение для оценки ошибки
- Можно статистически анализировать: "В 95% случаев когда confidence > 0.8, восстановление имеет ошибку < 5%"

#### 5. **Relevance**
- Для судебной экспертизы: восстановленные изображения часто используются как доказательства
- Confidence map позволяет судье оценить надежность этого доказательства
- Честная оценка неопределённости УВЕЛИЧИВАЕТ вес доказательства (показывает отсутствие bias)

## Техническое резюме для диссертации

### Основная архитектурная вклад

> **Hybrid Forensic Inpainting Generator (HFI-Gen)** комбинирует:
> 1. **Truncated U-Net** (4 уровня) для сохранения мелких структур судебных изображений
> 2. **Aggregated Contextual Transformations (AOT)** в bottleneck для контекстной агрегации
> 3. **Confidence Estimation Branch** для явной оценки уверенности в восстановленных регионах

### Основное научное утверждение

> *Confidence-aware инверсия позволяет эксперту различить надёжно восстановленные регионы от регионов с высокой неопределённостью, что критично для судебной допустимости восстановленного изображения как доказательства.*

### Ожидаемые результаты

- **LPIPS ↓ 15-20%** (лучшая перцептуальная близость к оригиналу)
- **FID ↓ 10-15%** (более натуральные восстановленные изображения)
- **Confidence Accuracy > 85%** (карта уверенности хорошо коррелирует с реальной ошибкой)
- **Text Recognition Accuracy ↑ 20-30%** (восстановленный текст читаем)
